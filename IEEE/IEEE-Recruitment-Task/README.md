# IEEE-Recruitment-Task

# 🔹 Task 2: Make Existing Website Responsive

## 📌 Objective
You will be provided with a static website built using HTML and CSS.  
Your task is to fork repo and convert it into a **fully responsive website** that works seamlessly across different screen sizes and devices.

---

## 🎯 Requirements

### ✅ 1. Responsiveness
- Ensure the website looks good on:
  - Mobile devices 📱
  - Tablets 📟
  - Laptops 💻
  - Large screens 🖥️
- Use proper breakpoints for different screen sizes

---

### ✅ 2. Layout & Alignment
- Fix any overflow or misalignment issues
- Maintain consistent spacing (padding & margins)
- Ensure elements are properly aligned across all devices

---

### ✅ 3. Flexible Design
- Use modern CSS techniques like:
  - Flexbox
  - Media Queries
- Avoid fixed widths where not necessary
- Make images and containers responsive

---

### ✅ 4. Navigation Handling
- Convert the navbar into a mobile-friendly version
  - Example: Hamburger menu (optional but recommended)

---

## ⭐ Bonus (Optional but Recommended)

- Improve UI/UX (better spacing, fonts, colors)
- Add a small interactive feature:
  - Navbar toggle
  - Smooth scrolling
  - Simple animations

---

## 📦 Submission Expectations

- Clean and well-structured code
- Proper folder structure
- Meaningful commit messages
- Deployed website link (Vercel / Netlify preferred)

---

## ⚠️ Notes

- Do not completely redesign the website  
- Focus on **making it responsive and improving usability**
- Keep the original structure intact while enhancing it
